# Local build and validation

The archive was reviewed and syntax-checked in an environment without the real libsodium development package. A temporary local API shim was used only to exercise control flow and tests; it is **not included** and is not a substitute for a real build.

Run the authoritative validation on a Linux machine with the actual dependencies:

```bash
sudo apt update
sudo apt install -y clang lld make pkg-config libsodium-dev libssl-dev python3

make clean
make release
make check

make sanitize
ASAN_OPTIONS=detect_leaks=1 ./symfrog512-sanitize --test-all

make full-suite
./symfrog_full_suite

python3 -m py_compile src/symfrog_inspect.py
```

Recommended additional checks:

```bash
clang++ --version
pkg-config --modversion libsodium openssl
ldd ./symfrog512
readelf -W -l ./symfrog512 | grep -E 'GNU_RELRO|GNU_STACK'
```

Expected hardening properties include PIE, a non-executable stack, RELRO, immediate binding, stack protection, fortified libc calls where available, and 64-bit file offsets.

## Compatibility check performed during revision

A legacy v1 file generated by the original source was successfully decrypted by the hardened v2 code in the local test harness. New encryption writes v2 only.

## Important limitation

Passing implementation tests does not prove P1024-v2 or the SymFrog mode cryptographically secure. Independent cryptanalysis remains necessary.
